Phase 3 — 진짜 QV(2차 비용+고정 예산)는 독점을 줄이는가

강도의 2차 비용을 고정 예산에 부과하는 *진짜* QV + 평판가중. Phase A가 못 한 "예산 있는 QV"를 구현하고, Sybil 취약점·방어를 단위 회계로 보인다.

✅ 실험 완료 + 단위테스트(275 통과). ★결과 = 실측(verify_qv.json, 0 불일치).
💡 배경지식 없이 한 번에. 진짜 이차투표(QV)는 "한 군데에 몰아 쓰면 값이 제곱으로 비싸지는 정해진 예산"이다. 욕심내서 한 번에 많이 가져가려 하면 예산이 순식간에 바닥나 스스로 손해를 본다. 그래서 독식이 저절로 억제된다. 이걸 넣었더니 독점이 절반으로 줄고 동시에 모두가 더 잘 끝냈다 — 이 프로그램에서 공정성·성공률이 둘 다 좋아진 유일한 방법. (다만 표본이 작아 통계적으로는 아슬아슬 — "유망하지만 더 큰 실험으로 확인 필요".) 독점(top-share)은 낮을수록, 후생(welfare, 완료율)은 높을수록 좋다.

1. Phase 3은 무엇인가 (설계)

보고서 향후 과제 중 "진짜 QV가 독점을 줄이나"를 측정한다. Phase A의 1/(1+o²) 캡은 예산이 없어 진짜 QV가 아니었다(리뷰 NO-GO). 진짜 QV(Weyl 2017)는 강도의 2차 비용을 고정 예산에 부과한다. 평판가중을 더하면 사용자가 물었던 "QV+평판 결합본"이 된다.

2. 메커니즘 · 수식 (구현)

구현: antigreedy/governance/qv.py(QuadraticVotingPolicy + 순수 회계 함수). recompute-on-read(단일 기록자 준수): 누적 지출을 turn_log에서 매번 재계산.

진짜 QV (2차 비용 + 고정 예산 + 평판가중): unit_cost(d) = d² / rep # 평판가중(저평판 비쌈); 무가중이면 d² spent_i = Σ_past d_t² / rep # 누적 2차 지출 (turn_log서 재계산) 남은 예산으로 살 수 있는 최대 = sqrt((B − spent)·rep) 요청이 이를 넘으면 캡(MODIFY) → 한 턴에 몰아쓰기 억제 Sybil (단위 검증, test_qv.py): 분할 비용 = total²/(k·rep) # k 신원으로 쪼개면 1/k (취약점) 방어: 신원-귀속 합산 예산 → total²/rep (k 무관, 이득 0)
  • 왜 예산이 핵심인가 — 2차 *곡선*만으론 부족(Phase A NO-GO). 고정 예산이 있어야 "한곳에 몰아쓰면 다른 데 못 쓴다"는 QV의 핵심 트레이드오프가 생긴다.
  • 평판가중(÷rep) — 저평판 에이전트는 같은 수요가 더 비싸 투표력이 약해진다 = 평판과 QV의 결합.
  • Sybil은 단위 테스트로 — 분할이 2차 비용을 1/k로 깎는 취약점과, 신원-귀속 예산 방어를 결정론적 회계로 보였다(LLM arm보다 명확·저비용).

3. 선행 연구와의 관계 (문헌 비교)

우리 qv.py를 QV 이론·실증 선행과 수식 단위로 대응시킨다. 전체 인용·웹검증은 docs/related_work.md §A(전부 [V] 표시).

선행 연구핵심 세팅 / 수식우리 qv.py 대응차이 (delta)
Lalley & Weyl, Quadratic Voting (2015/18) + Nash Eq. (1409.0264) 강도 진실 노출, n↑이면 점근 효율 — 단 비효율 ∝ 1/n unit_cost(d)=d² (2차 비용) n=4 소표본 → 1/n 비효율 영역 → 효과 약화·Holm 간발의 *이론적* 근거
Weyl, Robustness of QV (Public Choice 2017) "2차일 때만(iff) 강건 최적" — 예산·지불·price-taking 전제 qv_flat = 예산 있는 2차 → 정리 *적용 대상* Phase A의 1/(1+o²)는 예산·지불 없어 정리 비적용(NO-GO 근거)
Georgescu·Fox·Gautier·Wooldridge (arXiv 2409.06614) — 고정예산 다이슈 QV 비용 Σ_j m_ij² ≤ B (다대상·고정예산), 응용 = 기업/블록체인 거버넌스·설문·국민투표 예산 B + 2차 차감 골격 동일: spent=Σ d²/rep ≤ B ① 단일-대상(공유지 하나라 m_ij→d) ② ÷rep 추가 ③ 응용=발언권/자원(본문에 attention 배분 없음 — 웹검증 → 명시적 선행 부재)
FedQV (arXiv 2401.01168) — 평판-가중 예산 연합학습 집계에서 평판-가중 예산이 QV 이점을 *증폭* qv_rep: cost=d²/rep, cap=√((B−spent)·rep) "QV+평판"의 가장 가까운 기능적 선례. 단 도메인 다름(FL집계 vs LLM자원) — 우리 결과는 증폭 없음(예산이 먼저 binding)
Buterin·Hitzig·Weyl QF (1809.06421) / SBT·탈중앙사회 (2022) QF의 지배 공격 = Sybil(신원 분할); 비양도 신원(SBT)이 방어 토대 sybil_total_cost: 분할→total²/(k·rep), 신원-귀속→total²/rep 단위 회계로 취약점·방어 검증만 — 실 LLM 다신원 arm 미수행

한 줄 위치. 메커니즘(고정예산 2차 비용)도, 평판 가중(FedQV)도 선행이 있다. 그러나 이를 LLM 에이전트의 발언권/자원 거버넌스에 적용 + V6식 통제 검정으로 환원한 명시적 선행은 없다 — 좁지만 분명한 신규성은 *적용·위협모델·검정 설계*에 있다.
