Antigreedy — 발언권 커먼즈 A/B 시연

같은 에이전트, 같은 시드 — 오직 거버넌스 정책만 다릅니다 ❓ 도움말 & 정의 📑 거버넌스 문서

여러 AI 에이전트가 회의에서 하나의 예산을 나눠 쓰는 모습을 봅니다. 각 동그라미가 에이전트이고, 많이 말할수록 커집니다. 아래 막대는 커먼즈(commons) — 남은 공유 예산입니다. 이 나란히 보기의 목적: 왼쪽에서 "탐욕적인" 에이전트가 커먼즈를 고갈시키는 모습과, 오른쪽에서 거버넌스가 공정하게 유지하는 모습을 비교하는 것입니다.

커먼즈(Commons) = 모든 에이전트가 끌어 쓰는 공유 토큰 예산. 바닥나면 회의는 실패합니다. 발언량(Airtime) = 에이전트가 말한 양. 동그라미가 클수록 발언량이 많습니다. 판정(Verdict) = 거버넌스가 한 턴에 한 조치: allow(허용) · modify(축약) · deny(차단).
📖 용어 자세히 — 커먼즈 · 발언량 · 발언 점유 · 공정성 · 판정 (처음이면 펼쳐보세요)

🪙 커먼즈 (Commons · 공유 예산)

💡 회의실의 공용 화이트보드 잉크 같은 것. 모두가 같은 잉크를 나눠 쓰고, 한 사람이 다 써버리면 남들은 못 씁니다.

뜻 모든 에이전트가 말할 때마다 줄어드는 공유 토큰 예산.

화면 각 패널 아래 막대 — 가득(초록) → 적음(노랑) → 고갈(빨강). 합의 전에 0이 되면 회의 실패(고갈).

왜 이게 "공유지의 비극"의 그 공유지. 탐욕 에이전트가 여기서 예산을 말려버립니다.

🗣️ 발언량 (Airtime)

💡 회의에서 한 사람이 말한 시간의 총합.

뜻 각 에이전트가 실제로 전달한 누적 토큰.

화면 동그라미 크기로 표현 — 클수록 많이 말한 것.

📊 발언 점유 (Top share · 독점도)

💡 "가장 말 많은 한 명이 전체 발언의 몇 %를 가져갔나?"

뜻 최대 단일 에이전트의 발언 비중. 1.0 = 한 명이 100% 독점.

화면 메트릭의 "최대 발언 점유 84%". 이게 우리가 측정하는 greedy 지표 — 낮을수록 좋음.

⚖️ 공정성 (Jain 지수)

💡 피자 8조각을 8명이 똑같이 나누면 "공정". 한 명이 7조각 먹으면 "불공정".

뜻 발언량이 얼마나 고르게 나뉘었는지 0~1로 수치화 (1.0 = 완전 균등). 수학적으로 Jain's fairness index.

화면 "공정성(Jain) 0.99" — 0.8↑ 공정 · 0.6~0.8 다소 불균형 · 그 이하 독점적.

🚦 판정 (Verdict)

💡 회의 사회자가 각 발언에 내리는 결정.

allow(허용) 그대로 통과 · modify(축약) 너무 길어 잘라냄 · deny(차단) 발언권 박탈(배제).

화면 동그라미 색 + 피드의 색 단어로 표시.

🆚 베이스라인 vs 거버넌스

💡 약 안 먹은 그룹(대조군) vs 약 먹은 그룹.

왼쪽 베이스라인 = 무규제 대조군. 오른쪽 거버넌스 = 전략(정책) 적용.

왜 같은 에이전트·같은 조건, 오직 전략만 다름 → 두 패널의 차이가 곧 전략의 효과.

😈 greedy (탐욕 · 이기적 독점)

💡 회의에서 혼자만 계속 떠들어 남들이 말 못 하게 만드는 사람.

뜻 한 에이전트가 자기 이익(발언권)을 위해 공유 자원을 독점하는 행동. 이 테스트베드가 막으려는 대상.

🚫 배제 (Ostracism)

💡 계속 독점하던 사람을 잠시 발언에서 빼는 것 — 사회심리학의 저비용 처벌.

뜻 평판이 임계치 미만으로 떨어진 에이전트의 턴을 deny(차단). 점유가 떨어지면 평판이 회복돼 다시 참여.

화면 해당 에이전트가 빨강으로, 피드에 "deny … 차단됨".

하는 일: 양쪽을 나란히 실행해 비교합니다.
단일 실시간 회의 — 진행 중 거버넌스를 켜고 꺼서 효과를 봅니다.
한 에이전트가 몰래 축소 보고합니다 — 모니터가 잡아내는지 봅니다.
실시간 실행에 거버넌스를 즉시 적용/해제합니다.
범례 동그라미 색 = 최근 판정: allow 허용 modify 축약 deny 차단 / 기만 delay 지연 동그라미 크기 = 발언량 막대 = 남은 커먼즈 (가득 → 적음 → 고갈)

활성 정책 (거버넌스 측)
🧑‍🤝‍🧑 에이전트 편집 — 각 에이전트에 페르소나(성격) 부여

각 에이전트의 페르소나를 입력하면 프롬프트에 반영됩니다. 실 LLM에서는 행동이 바뀌고, Mock에서는 프롬프트에만 표시됩니다(턴 행을 클릭해 확인). 위의 에이전트 수를 바꾸면 목록이 자동 갱신됩니다.

🕘 실험 기록 — 지난 실험 (행을 클릭하면 다시 열어 재생)

실행한 모든 런이 저장됩니다(설정 + 전체 이벤트 로그). 행을 클릭하면 패널에 그대로 재생됩니다 — 일회성 시연을 재현 가능한 기록부로 바꿔줍니다.
